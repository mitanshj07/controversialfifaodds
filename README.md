### 🏆 The Payout Ladder

Equal scores **share rank AND share reward**. No tiebreakers based on who clicked faster or joined first. Pure football knowledge.

```
             🥇 Rank 1 → Largest slice of the pool
             🥈 Rank 2 → Solid reward
             🥉 Rank 3 → Still eating
             ...
             📊 Tied? → Same rank, averaged payout across positions
```

### ⏰ The Entry Deadline (Critical Game Theory)

The **first paid entry in any contest** triggers a **match-wide 20-second countdown**. Every contest for that match shares the same deadline. Why?

```
❌ Can't watch one contest's replay, then enter another contest with the answers
✅ All players face the same uncertainty at entry time
✅ Private rooms created during the window inherit the existing deadline
✅ Once the deadline hits: ENTRY_CLOSED for everyone
```

After the deadline, all room replays start synchronized. Late-joining members' rooms seek to the current authoritative match position, never starting from zero.

## 🔌 SOCKET CONTRACT — THE WIRE PROTOCOL

### *Every event. Every shape. Every acknowledgement.*

</div>

### 📤 Client → Server

```js
// 🎫 Start or resume a session
socket.emit('contest:session', {
  nickname: 'VARWhisperer',
  walletAddress: '6MS5...MJ5A',     // optional Solana pubkey
  participantId: 'abc123',           // optional, for resume
  resumeToken: 'xyz789'             // optional, for resume
});

// 📋 List available contests
socket.emit('contest:list', { participantId?, resumeToken? });

// 🔍 Preview a private contest by invite code
socket.emit('contest:lookup', { inviteCode: 'SQUAD42' });

// 🏗️ Create a private contest (auto-joins + charges creator)
socket.emit('contest:create', {
  name: 'The Inner Circle',
  entryCredits: 100,
  capacity: 10
});

// 🚪 Join a contest
socket.emit('contest:join', { contestId?, inviteCode? });

// 💳 Top up Demo Credits via Devnet SOL
socket.emit('wallet:buy_points', {
  signature: '5KtR...base58',
  packageId: 'pack_1',
  walletAddress: '6MS5...MJ5A',
  participantId: 'abc123',
  resumeToken: 'xyz789'
});

// 🏟️ Enter a jury room
socket.emit('room:join', { roomCode: 'JURY12', nickname: 'VARWhisperer' });

// 🗳️ Cast your verdict
socket.emit('vote:cast', { callId: 'call-opener-offside', choice: 'stands' });

// 🔄 Restart replay (non-contest rooms only)
socket.emit('replay:restart', {});
```

### 📥 Server → Client

| Event | When | Payload Shape |
|:------|:-----|:-------------|
| `contest:updated` | Contest state changes (new entry, status change) | Full contest summary |
| `room:state` | Room state update (scores, participants, your vote) | Full room state |
| `call:opened` | A Big Call drops — time to vote! | `{ callId, kind, title, detail, windowMs }` |
| `call:settled` | Verdict is in | `{ callId, result: 'stands' \| 'overturned' }` |
| `match:ended` | Full-time whistle | `{ outcome, homeScore, awayScore }` |
| `replay:restarted` | Demo replay reset | `{ generation }` |

### 🚨 Error Codes

```
INVALID_SESSION       → Your saved credentials expired. Re-auth with wallet + nickname.
INVALID_INVITE_CODE   → That code doesn't exist. Double-check with your squad.
CONTEST_NOT_FOUND     → This contest is gone. Refresh the lobby.
ALREADY_JOINED        → You're already in, legend.
CONTEST_FULL          → Too slow. Room's at capacity.
INSUFFICIENT_CREDITS  → Stack more DC before entering.
ENTRY_CLOSED          → The entry window has passed. Watch from the sidelines.
CONTEST_MEMBERSHIP_REQUIRED → You need to join the contest before entering its room.
RESTART_NOT_ALLOWED   → Contest rooms don't allow replay restarts. Server owns the feed.
OFFLINE               → "The match desk is reconnecting."
```

All errors follow `{ ok: false, code: 'ERROR_CODE', error: 'Human-readable message' }`.

## 🌐 LIVE MATCH INTEGRATION — TxLINE

### *When the demo isn't enough and you need REAL controversy*

</div>

The production adapter connects to **TxLINE SSE** for live match data:

```http
GET /api/scores/stream?fixtureId=...
Authorization: Bearer <guest-jwt>
X-Api-Token: <activated-api-token>
Last-Event-ID: <optional-resume-cursor>
```

Set these server-side env vars to activate:

```env
TXLINE_BASE_URL=https://txline-dev.txodds.com
TXLINE_API_TOKEN=your-api-token
TXLINE_FIXTURE_ID=12345
```

When the activated API token and fixture ID are present, the server
**auto-selects the live adapter** and creates/refreshes its guest JWT on the
server. Otherwise, it falls back to the demo replay. The adapter:

- 🔄 Deduplicates by `fixtureId:seq`
- 📍 Preserves SSE cursor for resume
- 💓 Handles heartbeats and reconnects
- 🟢 Opens calls from `var` events
- ✅ Settles only on confirmed `var_end` containing `Stands` or `Overturned`
- 🏁 Finalises rooms from `game_finalised` (`statusId=100`, `period=100`)

### 📡 Live Match Discovery

`GET /api/live-matches` proxies TxOdds/TxLINE fixture data to the frontend without exposing credentials:

```env
# TxLINE mode (preferred)
TXLINE_BASE_URL + TXLINE_API_TOKEN

# Legacy XML mode (fallback)
TXODDS_FIXTURES_URL + TXODDS_USER_ID + TXODDS_PASSWORD
```

For the Solana hackathon activation, Render setup, and pre-demo verification,
see [HACKATHON_SETUP.md](HACKATHON_SETUP.md). Keep all TxLINE credentials on
the server.

## 🔐 TRUST & PROOF BOUNDARY

### *We don't fake receipts.*

</div>

```
📜 WHAT WE DO:
   ✅ Settle jury from authenticated off-chain var_end message
   ✅ Generate deterministic SHA-256 reputation root after every settlement
   ✅ Label that root "devnet pending" until a real program publishes it
   ✅ Can later attach separate numeric stat proof

📜 WHAT WE DON'T DO:
   ❌ Claim VAR decisions are "verified on Solana"
   ❌ Present local hashes as on-chain facts
   ❌ Show simulated transactions as real ones
   ❌ Fake a blockchain receipt
```

> **The UI intentionally does NOT claim that a VAR decision is verified on Solana.** TxLINE's public stat-validation proves numeric score/card/corner state, not the `var_end` outcome. We keep it honest.

services:
